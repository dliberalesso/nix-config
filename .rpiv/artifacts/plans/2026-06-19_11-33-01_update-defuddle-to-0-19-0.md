# Defuddle 0.19.0 Implementation Plan

## Overview

This plan updates the custom `defuddle` package pin from 0.18.1 to 0.19.0 in the existing `buildNpmPackage` module. The chosen approach preserves the current package shape, overlay export, and `unify.home` installation path, and only refreshes the version plus the two fixed-output hashes required by Nix.

## Requirements

- Update the packaged `defuddle` version to `0.19.0`
- Preserve the existing `buildNpmPackage` structure in `modules/packages/defuddle.nix`
- Preserve overlay export through `overlayAttrs` and package export through `packages`
- Preserve `unify.home` installation of `pkgs.defuddle`
- Use exact, validated `src.hash` and `npmDepsHash` values for the new upstream release
- Keep the plan scoped to the package pin refresh only

## Current State Analysis

`defuddle` is packaged as a flat custom package module under `modules/packages/` and auto-imported into the flake through `import-tree`. The current derivation is already structured for version/hash refreshes, with `tag = finalAttrs.version` and nearby `lib.fakeHash` comments that document the expected maintenance workflow.

### Key Discoveries

- `modules/packages/defuddle.nix:10` exports `defuddle` through `overlayAttrs`, so downstream consumers resolve it as `pkgs.defuddle`
- `modules/packages/defuddle.nix:13-35` defines `defuddle` with `pkgs.buildNpmPackage`, currently pinned to `version = "0.18.1"`
- `modules/packages/defuddle.nix:20` ties the GitHub fetch tag directly to `finalAttrs.version`, so changing the version also changes the fetched upstream tag
- `modules/packages/defuddle.nix:23` and `modules/packages/defuddle.nix:27` are the two fixed-output hashes that must be refreshed for any upstream release bump
- `modules/packages/defuddle.nix:39-45` installs `pkgs.defuddle` in `unify.home`, so no extra host or profile wiring is needed
- `flake.nix:4-10` auto-imports `./modules` via `import-tree`, so the package module has no manual registration point to update
- `modules/packages/bmad-method.nix:9-22` follows the same `buildNpmPackage` + explicit hash pinning pattern and confirms the local convention to preserve the `lib.fakeHash` comments beside real hashes
- An isolated temporary `buildNpmPackage` derivation for defuddle `0.19.0` built successfully with `src.hash = "sha256-DtGfAu+Yv9AZVPXdf/UA0Fk2252v+WhznPyYNVCE3sQ="` and `npmDepsHash = "sha256-3YxwAyrQrxU0ADjyuQmOpxGRtJ9HgTDubvhH8Tr4aCA="`

## What We're NOT Doing

- Refactoring `modules/packages/defuddle.nix` into a multi-file package wrapper
- Adding `passthru.updateScript` or any automation for future version bumps
- Changing overlay wiring, package export names, or `unify.home` installation behavior
- Modifying hosts, profiles, or any package aggregation modules unrelated to `pkgs.defuddle`
- Adjusting upstream build behavior beyond what is required for the validated 0.19.0 pin

## Decisions

### Preserve the existing package shape

Decision: keep the current `pkgs.buildNpmPackage` derivation in `modules/packages/defuddle.nix:13-35` and update only the pinned values required for the upstream release.

Evidence:

- `modules/packages/defuddle.nix:13-35` already models the exact package shape needed for `defuddle`
- `modules/packages/bmad-method.nix:9-22` shows the same repo-wide convention for npm package pins built from GitHub tags

### Keep existing wiring untouched

Decision: preserve `overlayAttrs`, `packages.defuddle`, and `unify.home` exactly as-is.

Evidence:

- `modules/packages/defuddle.nix:10` exports the package through the overlay
- `modules/packages/defuddle.nix:39-45` installs `pkgs.defuddle` into the shared home profile
- `flake.nix:4-10` auto-imports the module, so no extra flake wiring exists to update

### Use validated fixed-output hashes in the plan

Decision: emit the exact hash values already confirmed by a successful isolated Nix build for `0.19.0`, rather than leaving fake placeholders in the plan.

Evidence:

- `src.hash` validated for `tag = "0.19.0"`: `sha256-DtGfAu+Yv9AZVPXdf/UA0Fk2252v+WhznPyYNVCE3sQ=`
- `npmDepsHash` validated through a successful isolated `buildNpmPackage` build: `sha256-3YxwAyrQrxU0ADjyuQmOpxGRtJ9HgTDubvhH8Tr4aCA=`
